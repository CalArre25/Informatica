Parte 1. Investigue qué es un sistema operativo (guía), tipos de sistemas operativos (guía) y compárelos en términos de ventajas, desventajas, seguridad, usabilidad y robustez (url).
-
Sistema Operativo (SO):
-
Es un software que gestiona los recursos de hardware y proporciona servicios comunes para los programas de aplicación. Actúa como intermediario entre el hardware y los programas de software, permitiendo la comunicación y la ejecución de tareas.

Tipos de Sistemas Operativos:
-
Sistema Operativo de Escritorio:
-
Ejemplos: Windows, macOS, Linux (Ubuntu, Fedora, etc.).

Ventajas: Amplia disponibilidad de software de terceros, interfaces gráficas amigables, soporte para una amplia variedad de hardware.

Desventajas: Pueden ser propensos a virus y malware, pueden requerir recursos significativos del sistema.

Sistema Operativo Móvil:
-
Ejemplos: Android, iOS.

Ventajas: Diseñados específicamente para dispositivos móviles, interfaces intuitivas, ecosistemas de aplicaciones robustos.

Desventajas: Menos personalización que los sistemas de escritorio, más cerrados en términos de configuración.

Sistema Operativo de Servidor:
-
Ejemplos: Windows Server, Linux Server (Ubuntu Server, CentOS, etc.).

Ventajas: Optimizados para servidores, alta estabilidad y escalabilidad, control sobre recursos del sistema.

Desventajas: Pueden requerir habilidades técnicas avanzadas para configurar y administrar.

Sistema Operativo Tiempo Real:
-
Ejemplos: QNX, FreeRTOS.

Ventajas: Diseñados para tareas críticas en tiempo real, alta confiabilidad y determinismo.

Desventajas: Menos flexibles en términos de uso general, pueden no ser adecuados para aplicaciones no críticas en tiempo real.

Comparación en Términos de Ventajas, Desventajas, Seguridad, Usabilidad y Robustez:
-
Ventajas:
-
Sistemas de Escritorio: Amplia disponibilidad de software y hardware, interfaces gráficas amigables.
Sistemas Móviles: Diseñados para dispositivos móviles, interfaces intuitivas, ecosistemas de aplicaciones robustos.
Sistemas de Servidor: Optimizados para servidores, alta estabilidad y escalabilidad.
Sistemas Tiempo Real: Diseñados para tareas críticas en tiempo real, alta confiabilidad.

Desventajas:
-
Sistemas de Escritorio: Pueden ser propensos a virus y malware, pueden requerir recursos significativos del sistema.

Sistemas Móviles: Menos personalización que los sistemas de escritorio, más cerrados en términos de configuración.

Sistemas de Servidor: Pueden requerir habilidades técnicas avanzadas para configurar y administrar.

Sistemas Tiempo Real: Menos flexibles en términos de uso general, pueden no ser adecuados para aplicaciones no críticas en tiempo real.

Seguridad:
-
Depende de la configuración y las prácticas de seguridad implementadas en cada tipo de sistema operativo.

Usabilidad:
-
Sistemas de Escritorio y Móviles: Suelen tener interfaces gráficas intuitivas y amigables.

Sistemas de Servidor: La usabilidad puede variar, dependiendo del nivel de experiencia del administrador.

Sistemas Tiempo Real: La usabilidad puede ser más técnica y específica para aplicaciones críticas.

Robustez:
-
Depende de la estabilidad y la capacidad de gestión de recursos del sistema operativo en cuestión.

-
Una imagen ISO es una representación digital de un disco óptico, como un CD o DVD. Contiene una copia exacta de todos los datos y la estructura de archivos presentes en el disco original. El formato ISO es ampliamente utilizado para distribuir software, sistemas operativos, juegos y otros contenidos digitales.
-
Cómo se Usa una Imagen ISO:
-
Montar la Imagen:
-
En sistemas operativos modernos, puedes montar una imagen ISO directamente haciendo doble clic en el archivo. Esto crea una unidad virtual que simula la presencia de un disco físico.

Grabar en un Disco Físico:
-
Si deseas crear un disco físico a partir de una imagen ISO, necesitarás un programa de grabación de discos (como Nero, ImgBurn o el propio software de grabación de Windows). Abre el programa, selecciona la opción para grabar una imagen y selecciona el archivo ISO.

Instalar un Sistema Operativo:
-
Para instalar un sistema operativo desde una imagen ISO, generalmente debes configurar tu computadora para arrancar desde el disco o unidad USB donde grabaste la imagen. Durante el proceso de instalación, se copiarán los archivos desde la imagen al disco duro.

Crear una Unidad USB de Arranque:
-
Puedes utilizar herramientas como Rufus, UNetbootin o la herramienta de creación de medios de Windows para crear una unidad USB de arranque desde una imagen ISO. Esto es útil para instalar sistemas operativos sin una unidad de disco óptico.

Extraer Archivos:
-
Puedes usar programas de descompresión como 7-Zip para extraer archivos de una imagen ISO sin necesidad de grabarlos en un disco.

Instalar Software o Juegos:
-
Algunas aplicaciones y juegos se distribuyen como imágenes ISO. Puedes usar un programa de emulación de unidades ópticas o montar la imagen directamente para acceder a su contenido.

Nota Importante:
-
Es importante recordar que solo debes utilizar imágenes ISO de fuentes legítimas y autorizadas. Descargar o distribuir contenido protegido por derechos de autor sin permiso puede ser ilegal en muchos países. Siempre adquiere o utiliza imágenes ISO de fuentes confiables y legales.

Parte 3. Investigue un poco sobre las distribuciones del sistema operativo GNU/Linux (guía). Descargue el archivo ISO del sistema operativo Linux Fedora (guía) o Zorin OS de 32 o 64 bits, según sea la arquitectura del procesador (CPU) de su computadora (x86_64, por lo general).
-
Distribuciones de GNU/Linux:
-
Las distribuciones de GNU/Linux son variantes del sistema operativo basado en el kernel de Linux, combinado con herramientas y utilidades del proyecto GNU. Cada distribución tiene su propia configuración, conjunto de software preinstalado y filosofía. Aquí te proporciono una breve introducción a algunas distribuciones populares:

Ubuntu:
-
Filosofía: Facilitar el uso del software libre para todos los usuarios, sin importar sus habilidades técnicas.

Características: Interfaz amigable (con entorno de escritorio GNOME por defecto), amplia comunidad y soporte.

Casos de Uso: Uso diario, desarrollo de software, servidores.

Fedora:
-
Filosofía: Ser líder en innovación de software libre y proporcionar una experiencia de usuario de vanguardia.

Características: Características de vanguardia, uso de software actualizado, gestión de paquetes DNF.


Casos de Uso: Desarrollo, estaciones de trabajo, servidores.

Debian:
-
Filosofía: Compromiso con el software libre y la estabilidad.

Características: Altamente estable, sistema de gestión de paquetes APT, amplia selección de arquitecturas y derivadas como Ubuntu y Linux Mint.

Casos de Uso: Servidores, estaciones de trabajo, sistemas embebidos.

Linux Mint:
-
Filosofía: Ofrecer una experiencia de usuario similar a la de sistemas operativos tradicionales.

Características: Interfaz familiar similar a Windows, fácil de usar, actualizaciones controladas.

Casos de Uso: Usuarios que migran desde Windows, uso diario.

Arch Linux:
-
Filosofía: Mantener la simplicidad y cercanía con el software.

Características: Sistema de inicio desde cero, paquetes actualizados continuamente, gran flexibilidad.

Casos de Uso: Usuarios avanzados, entusiastas de la personalización.

CentOS:
-
Filosofía: Priorizar la estabilidad y la seguridad.

Características: Basado en Red Hat Enterprise Linux (RHEL), ideal para servidores, ciclo de vida largo.

Casos de Uso: Servidores empresariales, entornos de producción.

OpenSUSE:
-
Filosofía: Proporcionar una distribución estable, fácil de usar y completa.

Características: Herramientas de administración YaST, opciones de escritorio variadas, entorno KDE prominente.

Casos de Uso: Desarrollo, estaciones de trabajo, servidores.

Kali Linux:
-
Filosofía: Enfocado en la seguridad y las pruebas de penetración.

Características: Herramientas de seguridad preinstaladas, orientado a pruebas de penetración y auditorías.

Casos de Uso: Seguridad informática, pruebas de penetración.

Cada una de estas distribuciones tiene sus propias fortalezas y se adapta a diferentes tipos de usuarios y casos de uso. La elección de una distribución dependerá de tus necesidades y preferencias personales.

Parte 5. Investigue los conceptos involucrados: bootear un equipo (guía), BIOS (guía), UEFI (guía), partición (física y lógica, de arranque) (guía), GPT y MBR (guía).
-
Bootear un Equipo:
-
Definición: Bootear un equipo significa iniciar o cargar el sistema operativo y otros programas necesarios en la memoria RAM desde un dispositivo de almacenamiento, como un disco duro o una unidad USB, para que la computadora pueda comenzar a funcionar.

BIOS (Basic Input/Output System):
-
Definición: La BIOS es un firmware integrado en la placa base de una computadora. Es el primer software que se ejecuta al encender la computadora y se encarga de realizar pruebas de hardware, cargar el sistema operativo y permitir la comunicación básica entre el sistema operativo y el hardware.

UEFI (Unified Extensible Firmware Interface):
-
Definición: UEFI es un estándar más reciente que reemplaza a la BIOS. Es un firmware más avanzado y versátil que ofrece una interfaz gráfica y más características que la BIOS tradicional. UEFI es compatible con discos duros grandes y tiene funciones de seguridad adicionales.

Partición (Física y Lógica, de Arranque):
-
Partición Física:
-
Definición: Es una división física de un disco duro en una o más áreas separadas que funcionan como unidades de almacenamiento independientes. Cada partición tiene su propio sistema de archivos y puede contener un sistema operativo.

Ejemplo: Un disco duro de 500 GB dividido en dos particiones de 250 GB cada una.

Partición Lógica:
-
Definición: Es una subdivisión de una partición extendida. No contiene un sistema de archivos propio y se utiliza para crear unidades de almacenamiento adicionales dentro de una partición extendida.

Ejemplo: Una partición extendida puede contener múltiples particiones lógicas, cada una con su sistema de archivos.

Partición de Arranque:
-
Definición: Es una partición especial en la que se carga el sistema operativo al arrancar la computadora. En sistemas Windows, se denomina "partición activa".

Ejemplo: En un disco duro, la partición que contiene el sistema operativo es generalmente la partición de arranque.

GPT (GUID Partition Table) y MBR (Master Boot Record):
-
GPT:
-
Definición: GPT es una forma moderna de organizar las particiones en un disco duro. Permite un mayor número de particiones y es compatible con discos duros de más de 2 TB. Utiliza un esquema de tabla de particiones GUID (Globally Unique Identifier).

Ejemplo: Se utiliza comúnmente en sistemas operativos más recientes y es necesario para aprovechar al máximo discos duros grandes.

MBR:
-
Definición: MBR es un esquema más antiguo para organizar particiones en un disco duro. Está limitado a 4 particiones primarias y tiene un límite de tamaño de disco de 2 TB.

Ejemplo: Aunque es menos flexible que GPT, todavía se utiliza en muchos sistemas operativos y discos duros más antiguos.

Estos conceptos son esenciales para comprender cómo se organiza y carga el sistema operativo en una computadora.
